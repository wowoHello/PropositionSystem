# 命題任務頁面 — 功能與設計說明書

> **文件版本**：v1.1  
> **最後更新**：2026-03-04  
> **對應頁面**：我的命題任務（cwt-list.html）

---

## 一、這個頁面是做什麼的？

「我的命題任務」是整個 CWT 命題工作平臺裡 **最核心的頁面**。命題教師在這裡完成所有跟出題有關的事情：新增題目、編輯題目、預覽考卷效果、把題目送審。

簡單來說，這就是老師們每天上線後待最久的地方。

---

## 二、畫面從上到下長什麼樣？

整個頁面由上而下大致分成這些區塊：

1. **頂部導覽列** — 左邊是平臺 Logo，中間可以切換不同梯次（專案），右邊顯示登入者姓名和登出按鈕
2. **麵包屑** — 顯示「首頁 > 命題任務」，讓使用者知道自己在哪
3. **頁面標題與說明** — 簡短描述這頁的用途
4. **分頁標籤** — 分成「命題作業區」和「審核結果與歷史」兩個頁籤
5. **篩選工具列** — 可以依題型、等級、狀態、關鍵字來篩選題目
6. **操作提示** — 藍色提示框，告訴使用者怎麼使用批次操作（可關閉）
7. **統計卡片** — 4 張卡片即時顯示各狀態的題目數量
8. **題目列表** — 表格呈現所有題目，可以勾選、預覽、編輯、刪除
9. **分頁器** — 翻頁用

另外還有兩個彈出視窗（不會一開始就看到）：
- **命題表單** — 全螢幕彈窗，用來新增或編輯一道題目
- **考試預覽** — 模擬考卷的感覺，讓老師看看題目印出來大概長什麼樣

畫面最底部還有一個隱藏的 **滑入式文字編輯器**（類似手機打字的鍵盤），打開後可以輸入富文本內容。

---

## 三、各區塊功能說明

### 3.1 專案切換器

放在導覽列正中央。點下去會展開一個下拉面板，裡面列出「進行中」和「已結束」的梯次，還有搜尋框可以快速找到想要的梯次。切換梯次後，整個頁面的資料會跟著變動。

### 3.2 分頁標籤

| 標籤 | 用途 | 對應的統計卡片 |
|------|------|-------------|
| 命題作業區 | 管理正在編輯、即將送審的題目 | 草稿 / 完成 / 送審 |
| 審核結果與歷史 | 查看已被審核過的題目最終結果 | 採用 / 改後再審 / 不採用 |

切換標籤後，下方的統計卡片、狀態篩選選項、以及表格列的顯示都會自動跟著調整。

### 3.3 篩選工具

提供 4 種篩選方式，可以任意組合使用：

| 篩選方式 | 說明 |
|----------|------|
| 題型篩選 | 下拉選單，選擇 7 種題型之一或「全部」 |
| 等級篩選 | 下拉選單，選項會根據選的題型自動改變（例如聽力題會變成難度一～五） |
| 狀態篩選 | 下拉選單，依當前分頁標籤列出對應狀態 |
| 關鍵字搜尋 | 輸入文字後即時過濾，按題目摘要內容搜尋 |

### 3.4 統計卡片

每組有 4 張卡片，即時顯示各狀態的題目數量。

**命題作業區的 4 張卡片**：
- **有效命題總計** — 點擊可看全部
- **命題草稿** — 點擊卡片可篩選草稿；點擊圖示可把勾選的題目**批次設為草稿**
- **命題完成** — 點擊卡片可篩選完成；點擊圖示可**批次設為完成**
- **命題送審** — 點擊卡片可篩選送審；點擊圖示可**批次送審**

**審核結果與歷史的 4 張卡片**：
- 有效命題總計 / 採用 / 改後再審 / 不採用（僅篩選用，不提供批次操作）

### 3.5 題目列表

表格呈現每道題目的基本資訊：

| 欄位 | 說明 |
|------|------|
| 勾選框 | 勾選後可做批次操作；表頭有全選功能 |
| 題目內容摘要 | 顯示這道題的精簡描述 |
| 等級 | 小標籤呈現（初級、中級…） |
| 題型 | 一般題目、長文題目 等文字 |
| 狀態 | 彩色標籤，不同狀態不同顏色 |
| 建立時間 | 首次建立的時間 |
| 最後修改時間 | 最近一次修改的時間 |
| 操作 | 動態按鈕，依狀態不同而不同（見下方） |

**操作按鈕的規則**：
- **可以編輯的題目**（草稿、完成、改後再審）→ 顯示「預覽 | 編輯 | 刪除」
- **被鎖定的題目**（已送審、不採用）→ 只顯示「預覽 | 檢視」，不能改也不能刪

**列的排序優先順序**：草稿最上面 → 改後再審 → 完成 → 送審 → 不採用最下面

---

## 四、命題表單（彈出視窗）

這是一個佔滿整個螢幕的大彈窗，用來新增或編輯題目。

### 4.1 頂部共用欄位

不管選什麼題型，最上面都會有這些欄位：

| 欄位 | 說明 |
|------|------|
| 題型選擇 | 下拉選單，切換後下方表單內容會跟著變 |
| 適用等級 | 下拉選單，選項會根據題型自動調整 |
| 命題者 | 自動帶入登入者姓名，不可修改 |
| 最後修訂日 | 自動記錄，不可修改 |
| 目前狀態 | 用彩色標籤顯示這道題當前的狀態 |

### 4.2 七種題型表單

每種題型的表單都是 **左側邊欄 + 右側主要內容** 的分欄排版。

---

#### ① 一般題目 / 精選題目（共用同一套表單）

**左側邊欄**：
- 主類（7 個分類可選：文字、語詞、造句標點…）
- 次類（依主類自動帶出對應選項）
- 難度（三選一按鈕：易 / 中 / 難）
- 正確答案（選 A/B/C/D，選完後右邊選項會出現綠色邊框標示）

**右側內容**：
- 題幹（點擊打開底部文字編輯器輸入，可附圖）
- 四個選項卡片（A/B/C/D，各自可輸入內容和附圖）
- 試題解析（說明為什麼選這個答案，無高度限制）

---

#### ② 長文題目

**左側邊欄**：
- 題型（引導寫作 / 資訊整合）
- 難度（易 / 中 / 難）

**右側內容**：
- 題目標題
- 文章內容（可附圖）
- 批閱說明

---

#### ③ 短文題組

**左側邊欄**：
- 主類／次類（固定為「文義判讀 / 篇章辨析」，不可改）
- 文體（文言文 / 應用文 / 語體文）
- 難度（易 / 中 / 難）

**右側內容**：
- **母題區**：題目標題 + 文章內容（可附圖）
- **子題區**：
  - 可以按「新增子題」按鈕，動態加入多道子題
  - 每道子題是一個可展開摺疊的手風琴卡片
  - 每道子題包含：題目內容、主向度選擇、能力指標選擇（會依主向度自動帶出）、試題解析
  - 子題填完後，手風琴標題會出現綠色勾勾✅

---

#### ④ 閱讀題組

**左側邊欄**：
- 文體（文言文 / 應用文 / 語體文）
- 難度（易 / 中 / 難）
- 正確答案下拉選單（跟目前展開的子題**雙向同步**）
  - 左邊改答案 → 右邊跟著變
  - 右邊改答案 → 左邊跟著變

**右側內容**：
- **母題區**：標題 + 文章內容（可附圖）
- **子題區**：
  - 可動態新增/刪除子題
  - 每道子題是手風琴卡片，含：題目 + ABCD 選項（可附圖）+ 正確答案下拉

---

#### ⑤ 聽力題目

**左側邊欄**：
- 核心能力 / 細目指標（依等級自動帶入，不可修改）
- 語音類型（對話 / 情境 / 陳述）
- 素材分類（生活 / 教育 / 職場 / 專業）
- 正確答案（A/B/C/D，選完右邊選項卡會標綠色邊框）

**右側內容**：
- 題目
- 語音內容（可上傳音檔）
- ABCD 四個選項卡片
- 試題解析

**特殊連動**：等級改變時，核心能力和細目指標會自動切換對應內容。例如選「難度一」→ 自動帶入「提取訊息 / 提取對話與訊息主旨」。

---

#### ⑥ 聽力題組

**左側邊欄**：
- 語音類型（對話 / 情境 / 陳述）
- 素材分類（生活 / 教育 / 職場 / 專業）
- 正確答案（跟展開中的子題雙向同步）

**右側內容**：
- **母題**：語音內容（可上傳音檔）
- **子題列表**：固定 **只有 2 題**，不能新增也不能刪除
  - 第 1 題：難度三 / 推斷訊息
  - 第 2 題：難度四 / 歸納分析
  - 每題的核心能力和指標已固定，各自含 ABCD 選項

---

### 4.3 表單底部按鈕

| 按鈕 | 動作 |
|------|------|
| 取消 | 關閉彈窗，不儲存 |
| 存為草稿 | 儲存題目，狀態設為「命題草稿」 |
| 儲存並確認 | 儲存題目，狀態設為「命題完成」 |

**檢視模式（唯讀）**：如果是從「檢視」按鈕進來的，所有欄位都會變灰色不可編輯，底部按鈕也會隱藏。

## 五、共用元件

### 5.1 底部滑入式文字編輯器

這是整個頁面最有特色的設計。所有需要輸入文字的地方（題幹、選項、解析…），都不是直接在欄位裡打字，而是：

1. 點擊輸入框（看起來像一個灰色的方塊，裡面有「點擊輸入…」提示）
2. 畫面底部會像手機鍵盤一樣**滑上來**一個文字編輯器
3. 編輯完畢按「完成」，內容就會填回去
4. 所有輸入框共用同一個編輯器，不會開啟多個

**編輯器配備**：
- Quill 富文本編輯器（支援粗體、斜體、底線、字體大小等）
- 中文標點符號快捷列：`，` `。` `、` `？` `！` `：` `；` `「」` `『』` `（）` `【】` `……` 共 12 種
- 即時字數統計

### 5.2 考試預覽

點擊列表中的「預覽」按鈕後，會跳出一個模擬考卷外觀的彈窗，讓老師看看題目在考卷上大概長什麼樣子。

- 7 種題型各自有獨立的呈現方式
- 單選題會顯示 (A)(B)(C)(D) 標準考卷格式
- 題組會顯示文章 + 編號子題
- 聽力題會顯示一個音檔播放器的佔位圖示

## 七、狀態定義與流程

### 狀態說明

| 狀態 | 顏色 | 意義 | 可否編輯 |
|------|------|------|---------|
| 命題草稿 | 🟠 琥珀色 | 老師正在編輯的題目 | ✅ |
| 命題完成 | 🟢 綠色 | 老師確認完成，但還沒送審 | ✅ |
| 命題送審 | 🔵 藍色 | 已提交給審題老師 | ❌（鎖定） |
| 改後再審 | 🟠 橙色 | 審題老師退回，要求修改 | ✅ |
| 採用 | 🔵 主色 | 審核通過 | ❌ |
| 不採用 | ⚫ 灰色 | 審核未通過 | ❌（鎖定） |

### 命題教師的操作流程

```
新增題目 → 命題草稿
    ↓ 確認完成
命題完成
    ↓ 批次或單筆送審
命題送審（鎖定）
    ↓ 審題老師審核...
    ├── 改後再審 → 老師修改後重新送審
    ├── 採用 ✅
    └── 不採用 ❌
```

## 八、設計風格

- **背景色**：淺灰 `#F0F2F5`（眼睛友善）
- **導覽列**：藍色 `#2563EB`
- **卡片**：白底 + 圓角 + 淡陰影
- **統計卡片**：滑鼠移上去會有浮動效果，增加互動感
- **狀態標籤**：邊框樣式（非實心），各狀態有獨立顏色
- **表單排版**：左 1/4 邊欄 + 右 3/4 主要內容，資訊分層清楚
- **選項卡**：選正確答案後，對應選項出現綠色邊框 + 勾勾圖示
- **子題手風琴**：點擊展開/收合，填完後標題會出現綠色勾勾

## 九、已知可改善項目

1. **資料存放方式不理想** — 目前所有題目資料是直接嵌在每一列的 HTML 屬性裡，未來遷移到 Blazor 後應改為從後端 API 取得資料，前端只負責顯示。

2. **七種題型的程式邏輯高度重複** — 每種題型都重複寫了「清空欄位」、「填入資料」、「收集資料」的邏輯。建議抽出共用的部分，各題型只需描述差異即可。減少將近 40% 的重複程式碼。

3. **全域函式過多** — 目前很多功能函式為了讓按鈕能直接呼叫而暴露在全域範圍。建議統一用事件委派的方式管理，減少全域污染。

4. **篩選效能可合併** — 目前篩選和統計計數各跑一次迴圈，可以合併成一次就好。

5. **Demo 資料應抽離** — 目前假資料直接寫在 HTML 裡面約 160 行。建議改由程式動態生成，這樣未來換成真的 API 回傳資料時，替換會更乾淨。
